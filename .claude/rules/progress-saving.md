# Progress.md 自动保存规则

## 触发条件

`totalTokensReminder: "countdown"` 注入的 `<total_tokens>N tokens left</total_tokens>` ≤60k 时自动触发，无需等待用户指令。

## 编写原则

- 精简的是废话，不是信息——每条都要对下一会话有用
- 不设硬性行数上限——宁可多 5 行也不丢关键事实
- 新会话前 2 分钟必须能靠这份文档完全接上上下文，不需重读代码

## 输出位置

项目根目录：`D:\Unity Responsity\Custom RP\Custom RP\Progress.md`

## 必须覆盖（缺一不可）

| # | 内容 | 说明 |
|----|------|------|
| 1 | 当前任务 | 在做什么、做到哪了、卡在哪 |
| 2 | 已完成 / 未完成 | 清晰划线，避免新会话重复做工或遗漏 |
| 3 | 关键决策 + 原因 | 为什么选 A 不选 B，当时的约束是什么 |
| 4 | 项目背景 | 1-2 句讲清项目、技术栈、目标 |
| 5 | 关键文件/路径 | 让新会话不需要重新搜索 |
| 6 | 已知坑/注意事项 | 容易踩的雷、已经踩过的雷 |
| 7 | 下一步行动 | 明确、可执行，新会话打开就能动手 |
| 8 | 引用 CLAUDE.md | 提醒新会话先读 `Assets/CLAUDE.md` |

```markdown

## 项目背景

## 当前任务
（在做什么、做到哪了、卡在哪）

## 状态
### 已完成
- ...

### 未完成
- ...

## 关键决策
| 决策 | 原因 |
|------|------|
| ... | ... |

## 关键文件
- `path/to/file` — 用途

## 注意事项 / 已知坑
- ...

## 下一步
1. ...
2. ...

## 参考
- 完整项目上下文：`Assets/CLAUDE.md`
```
